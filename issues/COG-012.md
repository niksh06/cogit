# COG-012: Integration experiment — drive Cogit from a real agent session

Priority: P0 (validates demand before more engineering)

## Problem

All MVP stories pass acceptance tests, but no real agent has ever used
Cogit. The core product hypothesis — "agents need committable reasoning
provenance" — is untested. ADR-0006 forbids optimizing before measured
pressure; the same logic applies to features.

## Task

Wire the prototype into one real agent workflow and record findings:

- Option A: a Claude Code hook (PostToolUse/Stop) that commits a thought
  per turn with tool observations as `tool_observation` claims.
- Option B: a minimal MCP server exposing `add-fact`, `commit-thought`,
  `blame-fact`, `log` to any MCP client.

Choose one, run it for a real multi-step task, then answer:

- Which commands were actually used? Which never?
- Did claim/assertion granularity fit, or was it too heavy per turn?
- Was blame-fact ever useful in anger (debugging a wrong belief)?
- What broke: performance, ergonomics, schema strictness?

## Acceptance criteria

- A working integration script/server exists under `prototype/` or a new dir.
- A findings report lands in the project reports hub.
- At least three backlog tickets are created or re-prioritized from findings.
