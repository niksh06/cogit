# COG-013: Rust `cogit-core` port reproducing frozen vectors

Priority: P1 (per ADR-0007; blocked conceptually by COG-011 outcome)

## Task

Implement `crates/cogit-core` per ADR-0007 with the object store slice
first: canonical JSON, preimage, SHA-256 IDs, zlib storage, schema
validation, object-level verify.

## Contract

- Must reproduce every object ID in `prototype/vectors/object-vectors-v1.json`
  byte-for-byte.
- Canonicalization profile is fixed by ADR-0010 (code-point key order,
  minimal escaping, integers only ≤ 2^53−1); do not adopt a crate default
  that deviates.
- Dependency policy per ADR-0007: reliable crates allowed, no storage
  engines, no async runtime, no daemon.

## Acceptance criteria

- `cargo test` includes a golden test over the shared vectors file.
- Corruption fixtures (zlib, header, size, hash-path, non-canonical body)
  fail reads exactly as in `prototype/tests/test_store.py`.
- Behavior parity notes documented for any intentional divergence.
