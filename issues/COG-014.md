# COG-014: Atomic symbolic-HEAD updates

Priority: P1 (correctness under concurrency)

## Problem

Branch ref updates use lockfile + old-target check (`refs.update_ref`),
but HEAD writes (`refs.write_head`) use locked replacement WITHOUT an
old-value check. Two concurrent processes doing checkout/detached-commit
can interleave: the second overwrites the first without detection. Branch
tips are safe; HEAD position and its reflog can lose a movement.

## Task

- Extend the HEAD write path with an expected-old check under the lock
  (compare full HEAD content: symbolic target or detached oid).
- Surface mismatch as `ConcurrentUpdateError` (exit 4), consistent with
  branch refs.
- Detached `commit-thought` must pass the previous HEAD content as
  expected-old.

## Acceptance criteria

- A failure test simulating a concurrent HEAD move between read and write
  (pre-created `HEAD.lock`, and stale-expected-old) exits 4 and leaves HEAD
  plus `logs/HEAD` consistent.
- `docs/spec/repository-layout-v1.md` locking section mentions HEAD
  explicitly.
