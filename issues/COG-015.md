# COG-015: Negation-aware merge and commit checks (invariants 24–25)

Priority: P1

## Problem

Invariants: "a negated claim must explicitly link to the claim it negates"
(24) and "activating a negated claim requires removing the original active
assertion with reason `refuted`" (25). The object schema supports
`claim.negates`, and `remove-fact --reason refuted` exists, but nothing
ENFORCES invariant 25: today a commit can activate an assertion about a
negated claim while the original assertion stays active — a contradictory
mindset with no conflict recorded.

## Task

- `commit-thought`: reject a new mindset that simultaneously contains an
  assertion about claim X and an assertion about a claim that `negates` X,
  unless the original is being removed in the same commit with reason
  `refuted`.
- `merge`: treat "ours activates claim X, theirs activates negation of X"
  as a conflict on the negated pair, not an auto-union.
- Extend `docs/spec/claim-assertion-examples.md` Example 5 with the
  enforced flow.

## Acceptance criteria

- Unit test: contradictory commit is rejected with a clear message.
- Merge test: negation pair produces a conflict entry, never a silent union.
- `verify` gains a warning for pre-existing contradictory mindsets.
