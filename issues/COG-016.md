# COG-016: License and open-source posture (OQ-004)

Priority: P1 (blocks any publication)

## Task

Decide license and contribution model before the repository is published
anywhere.

Options to evaluate:

- MIT / Apache-2.0 (max adoption; Apache adds patent grant — usual pick
  for infra tools);
- MPL-2.0 (file-level copyleft middle ground);
- BUSL-style source-available (protects a future hosted product, hurts
  open-source adoption).

Note: `git/` reference tree is GPLv2; it must never be vendored into the
published product repo (see COG-017) regardless of the license chosen.

## Acceptance criteria

- LICENSE file at repo root; OQ-004 closed with a CQ entry.
- README states the license.
- Contribution note (even one paragraph) exists.
