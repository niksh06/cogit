# COG-017: Repo housekeeping — exclude `git/` reference tree, init product VCS

Priority: P1

## Problem

- `git/` is a 366 MB copy of the upstream Git source used as design
  reference. It is GPLv2, dwarfs the product (docs + prototype < 1 MB),
  and must not ship in the product repository.
- The cogit project itself is not under version control — ironic for a
  VCS product, and risky: doc/prototype history is unrecoverable.

## Task

1. Keep `git/` out of the product VCS: `.gitignore` entry (done) — or move
   the tree out of the repo entirely and replace ADR references
   `git/<path>` with upstream URLs (preferred, needs owner decision since
   ADR reference paths break).
2. `git init` the repository, initial commit of docs + prototype +
   backlog. Requires owner confirmation per workflow (outward-facing,
   creates durable history).

## Acceptance criteria

- Product repo history exists; `git status` clean; `git/` not tracked.
- ADR references remain resolvable (either local tree still present and
  ignored, or links rewritten to upstream).
