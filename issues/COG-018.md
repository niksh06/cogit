# COG-018: Notes-like annotations without rewriting objects

Priority: P1 (US-017, roadmap Phase 4; closes OQ-007)

## Problem

Human review, eval results, and policy flags need to attach to committed
thoughts and facts after the fact. Object IDs must not change; the raw
object view stays authoritative (invariant 19).

## Decisions

See ADR-0012: sixth object type `annotation`, append-only chain per typed
namespace under `refs/notes/<namespace>`, default namespace `notes`;
corrections are new annotations; reads are linear chain walks.

## Acceptance criteria

- `cogit annotate <target> --message <text>` annotates thoughts,
  assertions, and claims; the target object is byte-identical afterwards.
- Annotation history is versioned: two annotations on one target read back
  newest-first with intact chain parents; every notes-ref move is
  reflogged.
- `cogit annotations <target>` filters by target across namespaces (or one
  with `--namespace`); `cogit log --annotations` displays annotations
  inline under each thought.
- `verify` checks: notes refs point to annotation objects, chain parents
  and targets exist, object namespace matches its ref.
- Secrets in annotation bodies are rejected, not stored.
- Vectors file gains exactly one `annotation` entry; the five existing
  vectors stay byte-identical.
