# COG-019: `log --introduced-fact` / `--removed-fact` queries

Priority: P1 (US-018, roadmap Phase 4)

## Problem

`blame-fact` answers only "first introducer". Auditing belief churn needs
the full event list: every thought that introduced or removed a fact in the
selected ancestry (a fact can be removed and re-introduced).

## Decisions

- Event semantics (exact-ID, no semantic matching):
  - *introduced* at thought T: fact in mindset(T) and in no parent mindset;
  - *removed* at thought T: fact absent from mindset(T) and present in at
    least one parent mindset.
- Scope is the selected ancestry (ref or thought, default `HEAD`), same as
  `log`. Scanning all refs is future work and must be requested explicitly
  when added.
- Output order matches `log`: newest first.

## Acceptance criteria

- `cogit log --introduced-fact <id>` and `--removed-fact <id>` list matching
  thoughts with the usual log fields plus the event kind; `--json` works;
  abbreviated IDs accepted.
- Re-introduction after removal yields two `introduced` events.
- Flags are mutually exclusive with `-g` and with each other.
- CLI contract documents exact-ID semantics.
