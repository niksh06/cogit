# COG-021: `bisect-thought` — find where reasoning first went wrong

Priority: P1 (US-020, roadmap Phase 4; closes OQ-009)

## Problem

When an agent discovers a bad conclusion, it needs the first thought where
an oracle starts failing — binary search over reasoning history, the
`git bisect` of cognition.

## Decisions (oracle contract, closes OQ-009)

- Invocation: `cogit bisect-thought --good <id> --bad <id> --run <command>`.
- Candidates: thoughts reachable from bad but not from good, topologically
  ordered oldest-first; good must be an ancestor of bad. MVP assumes the
  predicate is monotonic along this order (primary use: linear histories).
- Probes are NON-MUTATING: no checkout. The oracle receives env vars
  `COGIT_THOUGHT`, `COGIT_MINDSET`, `COGIT_REPO` and inspects state via
  read-only commands (`cogit facts "$COGIT_THOUGHT"` etc.).
- Oracle exit codes (git-bisect compatible): `0` good, `125` skip/unknown,
  any other code `< 128` bad; `>= 128` (signal) aborts the bisect.
- Skips: the nearest untested candidate in the current range is probed
  instead; when skips leave more than one possible first-bad thought, the
  result is the remaining candidate range, reported as inconclusive
  (git-bisect's "could be any of" semantics).
- Replayable log: one line per probe `<thought-id> <verdict>`, printed and
  optionally written via `--log <file>`. The repository layout is not
  extended; bisect keeps no persistent state in `.cogit/`.

## Acceptance criteria

- Automated run finds the first bad thought on a linear history.
- Skip verdicts are honored; all-skipped ranges report inconclusive.
- The log allows replaying verdicts without re-running the oracle.
- OQ-009 closed with a CQ entry; CLI contract documents the command.
