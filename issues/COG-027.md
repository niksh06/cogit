# COG-027: Shorthand fact input for `add-fact`

Priority: P1 (from COG-012 findings)

## Problem

Composing a full claim+assertion JSON document is ~10 lines per fact. In the
dogfood session every fact was hand-built JSON; in the Claude Code hook the
boilerplate is generated code. Agents will avoid a journal that costs this
much per belief.

## Task

Add flag-based shorthand that builds the document:

```sh
cogit add-fact --kind agent_decision --subject cogit:COG-015 \
  --predicate approach --object "negation groups" \
  --source agent:session-x --confidence 9500 --actor fable [--method type=design_decision]
```

- Timestamps default to now (UTC); `--asserted-at` overrides.
- JSON input stays supported and canonical; shorthand is sugar only.
- `--negates <claim-id>` builds a negating claim.

## Acceptance criteria

- Shorthand and equivalent JSON produce identical object IDs.
- CLI contract documents both forms.
- Tests cover defaulting and `--negates`.
