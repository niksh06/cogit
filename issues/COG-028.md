# COG-028: `cogit facts` / `cogit show` porcelain

Priority: P1 (from COG-012 findings)

## Problem

There is no way to see WHAT the agent currently believes without manually
chaining `log --json` -> `cat-object <mindset>` -> `cat-object <assertion>`
-> `cat-object <claim>`. During dogfood this chain caused a real mistake:
the operator picked the wrong assertion ID out of a lexicographically
sorted mindset list and blamed the wrong fact. The provenance data was
correct; the porcelain was missing.

## Task

- `cogit facts [<ref-or-thought>]` — table of active facts for a thought:
  short assertion ID, claim subject/predicate/object, confidence,
  source type, introducing thought (optional `--blame` column).
- `cogit show <thought-id>` — thought header plus its facts (like `git show`).
- Both support `--json`.

## Acceptance criteria

- `cogit facts` output is enough to pick the right assertion ID for
  `blame-fact`/`remove-fact` without `cat-object`.
- Works for detached HEAD and named refs, including anchors.
- Tests cover text and JSON output.
