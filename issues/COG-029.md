# COG-029: MCP server for Cogit

Priority: P0 for the "real usage" phase

## Problem

Real agent adoption needs Cogit inside the agent's tool loop, not behind a
shell. MCP is the standard transport: one stdio JSON-RPC server makes Cogit
available to Claude Code, Claude Desktop, and any MCP client. The hooks
integration (COG-012) covers passive journaling; MCP covers ACTIVE use —
the agent deciding to commit a belief, blame a fact, or bisect its own
reasoning.

## Decisions

- Zero-dependency Python stdlib server (`prototype/integrations/mcp_server.py`),
  stdio transport, MCP protocol (initialize/tools list/tools call).
- Tool surface mirrors the agent-facing porcelain, JSON in/out:
  `add_fact` (shorthand fields), `remove_fact`, `commit_thought`, `facts`,
  `recap`, `log`, `blame_fact`, `branch`, `checkout`, `merge`, `resolve`,
  `anchor`, `annotate`, `annotations`, `status`, `verify`, `bisect_thought`.
- Repository path comes from server config/env (`COGIT_REPO`), one server
  per journal.
- Destructive surface stays constrained per ADR-0009 (no prune, no expire
  via MCP in v1).

## Acceptance criteria

- Server passes an MCP handshake and lists tools with JSON schemas.
- An end-to-end scripted client can add a fact, commit, and blame it.
- Registered and exercised in a real Claude Code session; findings recorded.
