# COG-030: add-fact --commit and stdin input (model-review finding)

Priority: P1

## Problem (from the model-perspective review, 2026-07-02)

The dominant agent gesture is "record one belief NOW": stage + commit as
one action. Today that is two CLI calls per belief; in hooks and MCP
handlers the extra round-trip doubles latency and code. Also, hook
pipelines naturally hold the fact document on stdin, but `add-fact` only
accepts an argument or a file.

## Decisions

- `cogit add-fact ... --commit [--message <m>] [--author <a>]` stages and
  immediately commits a thought. Default message is derived from the
  claim (`<kind>: <subject> <predicate>`), default author is the
  assertion's actor.
- `--commit` refuses when other facts are already staged or a merge is in
  progress — a micro-commit must not swallow unrelated staged state.
- `cogit add-fact -` reads the JSON document from stdin.

## Acceptance criteria

- One invocation produces claim, assertion, mindset, and thought; ref and
  reflogs move exactly as with the two-step flow.
- Dirty-index refusal is tested.
- stdin form round-trips byte-identical object IDs.
