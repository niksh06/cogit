# COG-031: cogit recap — belief-state digest for context recovery (model-review finding)

Priority: P1

## Problem (from the model-perspective review, 2026-07-02)

When an agent resumes work (new session, compacted context), the question
is "what changed since <anchor/thought> and what do I believe now?".
Today that requires log + diff + facts + cat-object choreography. Context
recovery is THE moment an agent-facing tool must optimize for.

## Decisions

- `cogit recap <from> [<to=HEAD>]` prints a compact digest:
  - thoughts between `from` and `to` (count + one-line each);
  - net belief changes: added/removed facts with decoded claim content
    (facts-style rows), not raw IDs;
  - current position (branch/detached, merge state).
- `from` accepts anchors — `cogit recap task-understood` is the primary
  use.
- `--json` for MCP/scripting.

## Acceptance criteria

- Digest of a 3-thought span shows all thoughts and the correct net
  added/removed facts with claim text.
- Anchor as `from` works; `from` not an ancestor of `to` is a user error
  with a hint to run diff instead.
- Output is a single screen for typical spans.
