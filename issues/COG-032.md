# COG-032: Claim-modeling cookbook (model-review finding)

Priority: P1 (docs)

## Problem (from the model-perspective review, 2026-07-02)

The subject/predicate/object claim shape is the product's semantic
backbone, but nothing teaches an agent to use it well. The likely failure
mode: everything becomes `predicate: "observation", object: "<text blob>"`
— at which point claim identity, negation, and claim-level merge stop
meaning anything. The schema cannot prevent this; guidance must.

## Task

Write `docs/claim-modeling.md`:

- one proposition per claim; the object is a value, not a sentence;
- how to choose subject granularity (entity URIs: `file:...`, `user`,
  `cogit:COG-015`);
- confidence calibration guidance (bps bands for observed/inferred/heard);
- when to add a rival assertion vs a negating claim vs a removal;
- qualifiers vs new claims; anti-patterns with corrected examples;
- what does NOT belong in Cogit (chat logs, tool noise — link non-goals
  and the journal-growth policy from COG-012 F5).

## Acceptance criteria

- Every anti-pattern has a bad/good example pair.
- README links the cookbook; hook and MCP docs point to it.
