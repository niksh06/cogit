# COG-033: Multi-process index safety (model-review finding)

Priority: P2

## Problem (from the model-perspective review, 2026-07-02)

Refs are protected by lockfiles and old-target checks (US-006), but
`index.json` is read-modify-write without a lock: two agent processes
staging into the same repository can lose each other's staged facts
(last-writer-wins on the whole file). Single-agent use is safe; the
multi-process story stops at the ref layer.

## Options to evaluate

1. Lockfile around every index read-modify-write (simplest; serializes
   staging).
2. Per-actor index files merged at commit (lock-free staging, more moving
   parts).
3. Declare the index single-writer by contract and detect violation via a
   session marker (cheapest, honest).

## Acceptance criteria

- Decision recorded (ADR or CQ) with a failure test demonstrating the
  current hazard first.
- Layout spec updated to state the concurrency contract for index.json.
