# COG-034: Live MCP session findings

Priority: P0 (usage phase; follow-up to COG-029)

## Problem

The MCP server passes a scripted end-to-end client (handshake, tool
listing, full workflow), but no real agent session has used it yet. This
is the actual experiment COG-012 pointed at: does an agent, given these
tools mid-task, record beliefs worth auditing?

## Task

1. Owner registers the server (one command, see
   `prototype/integrations/README.md`) and restarts the session:

   ```sh
   claude mcp add cogit -e COGIT_REPO=$HOME/.cogit-journal/cogit \
       -- python3 /ABS/PATH/prototype/integrations/mcp_server.py
   ```

2. Run one real multi-step working session where the agent uses the tools
   organically (recap at start, add_fact for decisions, anchor milestones,
   blame/bisect when something is wrong).
3. Record findings in the reports hub, mirroring the COG-012 format:
   which tools were used/ignored, granularity fit, journal growth,
   friction; spawn/re-prioritize tickets from findings.

## Acceptance criteria

- A findings report exists with at least one full session's journal.
- Backlog updated from findings.
- Decision recorded: is the fact-selection policy (COG-012 F5) worth an
  OQ/ADR now?
