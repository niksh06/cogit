# COG-036: Belief queries and no-argument recap

Priority: P1 (COG-034 live findings)

## Problem

In a shared, long-lived journal `facts` dumps everything, and `recap`
requires knowing an anchor name — the resuming agent doesn't. First live
session also showed `recap` from the anchor you are standing on returns
empty lists with no explanation.

## Decisions

- `facts` gains filters: `subject` (exact, or prefix with trailing `*`),
  `predicate` (exact), `project` (matches the `project` qualifier,
  COG-037). NOT semantic search — exact URI matching only (ADR-0002
  boundary intact). Fact rows now include `qualifiers`.
- `recap` source becomes optional: default is the NEWEST anchor (by
  anchor object `created_at`); with no anchors, the root thought. The
  result reports `from_anchor` (name or null) and `same_point`
  (from == to) so an agent never guesses.

## Acceptance criteria

- `facts --subject 'proj:cogit/*'` and MCP equivalents return only
  matching rows; unknown filters return empty, not errors.
- No-arg `recap` picks the newest anchor; `same_point: true` when HEAD is
  the anchor. Both runtimes + interop step.
