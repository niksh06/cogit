# COG-037: Project convention for shared journals

Priority: P1 (COG-034: one journal for all projects)

## Problem

The owner runs ONE journal for every project and agent. Subjects from
different projects mix; `beliefs about X` becomes ambiguous.

## Decisions

- Project identity lives in the `project` QUALIFIER (not a subject
  prefix): subjects stay natural entity URIs (`cogit:COG-015`,
  `api:/orders`), and claim identity still distinguishes projects because
  qualifiers are part of the canonical claim.
- MCP `add_fact` gains optional `project`; CLI gains `--project` sugar
  (equivalent to `--qualifier project=<name>`). `facts` filters by it.
- Cookbook rule: in a shared journal, ALWAYS set project.

## Acceptance criteria

- Same subject+predicate+object with different `project` qualifiers are
  DIFFERENT claims (identity test).
- Cookbook updated; MCP/CLI documented. Both runtimes.
