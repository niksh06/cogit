# COG-039: Belief-recovery benchmark — does the journal beat markdown, with data

Priority: P0 (owner request: close the "markdown is enough" question with data)

## Problem

The honest open question from live usage (COG-034 discussion): for belief
recovery after context loss, is a structured journal measurably better
than the disciplined markdown notes an agent keeps anyway — or is Cogit a
projection of human tooling with no efficiency gain? Opinions exist;
data does not.

## Experiment design

Synthetic but realistic working sessions with known ground truth; three
recording media; identical probes; fresh context-free agents as readers;
mechanical grading.

- **Generator** (`prototype/benchmarks/belief_bench.py`, stdlib + cogit
  API, seeded): each session is a stream of events over typed
  subject/predicate slots — initial assertions, supersedes, refutations
  (negation flow), milestones, plus noise events. Ground truth = final
  belief state + full revision history.
- **Recorders** (the three competitors, each written idiomatically, no
  strawmen):
  1. `journal/` — cogit repository: micro-commits, staged
     supersede/refute flows with reasons, anchors at milestones.
     Noise skipped (cookbook Rule 7).
  2. `markdown/notes.md` — what a disciplined agent keeps: a
     current-state table updated in place PLUS an append-only decision
     log with milestone markers. Noise skipped.
  3. `transcript/transcript.log` — append-only raw log, noise included
     (the "no discipline" baseline).
- **Probes** (10 per session, 5 classes): P1 current value; P2 source
  (type:uri) of the current belief; P3 revision history (was V ever
  believed; refuted/superseded/never; replacement); P4 delta since the
  last milestone; P5 confidence band + basis taxonomy
  (observation/statement/inference).
- **Readers**: one fresh read-only agent per (session, medium), sees ONLY
  that medium (cogit reader uses the CLI, others read the file), answers
  probes as strict JSON.
- **Metrics**: accuracy per probe class per medium (P4 by F1); medium
  size in bytes (read-cost proxy); recorder write ops and bytes
  (write-cost side).

## What results would mean

- Markdown ≈ journal on P1..P5 → structural value is multi-agent/audit
  only; solo pitch dies, product pivots harder to coordination and
  forensics.
- Journal wins on P2/P3/P4 while markdown holds P1 → media are
  complementary layers; the pitch is "ledger + notes", not "ledger vs
  notes".
- Transcript competitive anywhere → both structured media are
  over-engineering for that class.

## Acceptance criteria

- Reproducible harness committed (`generate` + `grade` subcommands,
  seeded), covered by a smoke test (generation invariants, cogit repo
  verifies, grader scores a perfect answer sheet at 1.0).
- One full run (>= 4 sessions x 3 media x 10 probes) with real
  context-free readers; findings report in the reports hub with the
  per-class accuracy table and an explicit verdict against the
  hypotheses above.
