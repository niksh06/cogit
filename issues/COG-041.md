# COG-041: Belief-recovery benchmark at scale (long horizon)

Priority: P2 (follow-up to COG-039; blocks any "helps solo agents" claim)

## Problem

COG-039 run 1 established that at single-session scale (~20 events,
~9 KB) a strong reader recovers beliefs perfectly from markdown AND from
a raw transcript, cheaper than from the journal. That run could not test
the regime the journal is designed for: histories that do not fit a
reader's working context, where `recap` stays O(delta) while transcript
and decision-log grow without bound.

## Design

- Scale the generator: 200-400 events per history, multi-"session"
  streams concatenated (10+ segments with milestones), so transcript and
  markdown decision log reach hundreds of KB while the current-state
  table and `recap <last-anchor>` stay small.
- Same probe classes; P3/P4 probes deliberately target beliefs revised
  MANY segments ago (deep history) and deltas across segment boundaries.
- Add a reader-budget axis: run readers with a capped tool/token budget
  and measure accuracy vs budget (the "cost frontier"), not just
  unlimited-budget accuracy.
- Fix COG-040 first — otherwise the journal's P3 handicap is a known
  rendering bug, not a medium property.

## Hypotheses to test

- H1: transcript accuracy degrades first as size exceeds comfortable
  context; markdown's decision log follows; journal recap/facts stay
  flat.
- H2: journal reader cost grows sublinearly with history length;
  file-media reader cost grows linearly.
- H0 (kill criterion): if journal shows no accuracy or cost advantage
  even at 10x scale, the solo-agent value proposition is dead at any
  scale — the product story is multi-agent/audit only.

## Acceptance criteria

- One full run at >= 10x COG-039 event volume, published like the
  COG-039 findings report, with an explicit H0/H1/H2 verdict.
