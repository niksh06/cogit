# COG-043: Session-start re-anchor hook (coverage begins with cheap resume)

Priority: P1 (COG-041 finding: the journal loses only where nothing was
written or nothing was read at resume)

## Problem

Re-anchoring on the journal is voluntary: the agent must remember to call
`recap`/`dump` when a session starts or resumes after a compact. In
practice this is the first thing skipped — the protocol relied on
discipline, and COG-041 measured what discipline costs.

## Decision

Extend `claude_code_hook.py` with a `session-start` mode wired to the
Claude Code `SessionStart` hook: it prints a compact digest built from
`dump` (active belief count, delta since the newest anchor, the last 8
thought messages, and a pointer to the `dump` MCP tool) directly into the
new session's context. `COGIT_PROJECT` scopes the digest for shared
journals. Failures never break the session (exit 0, COG-012 contract).

## Acceptance criteria

- `session-start` on an empty journal prints a one-line note; on a live
  journal prints beliefs count, since-anchor delta, recent messages.
- Covered by `tests/test_hook.py` (also the first automated tests for
  the COG-012 capture modes); wiring documented in integrations README.
