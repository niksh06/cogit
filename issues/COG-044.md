# COG-044: Selective auto-capture — closing the coverage gap

Priority: P1 (the main open finding of COG-041/COG-034; needs owner
decisions before implementation)

## Problem

Both benchmark rounds converge on one weakness: the journal is only as
good as its write coverage. Passive capture (COG-012 hook) records
everything and drowns signal in tool noise; active journaling (COG-034
protocol) is honest but sparse — the agent records ~1 fact per milestone
because writing costs attention mid-task. Real losses in COG-041 Track B
were absences, not misreads.

## Candidate mechanisms (to decide, not yet build)

1. **Event-selective hook capture**: extend the PostToolUse hook to
   record only durable event shapes — git commits (message -> decision
   fact), test-run outcomes (pass/fail counts), file creations of
   reports/specs. Deterministic, no LLM in the loop, low noise; misses
   everything else.
2. **Turn-digest extraction**: on Stop, feed the turn's tool summaries to
   a small model that proposes claim-shaped facts for the agent to
   confirm next turn. Highest coverage, adds cost/latency and a
   confirmation loop; needs harness support.
3. **Protocol + affordance only**: keep manual journaling but make it
   one-call cheap everywhere (dump on resume — COG-043 — plus a
   `commit_thought`-with-facts batch MCP tool), and measure whether
   coverage improves organically.

## Entry criteria / measurement

- Re-run the COG-041 real-history track after 1-2 weeks of the chosen
  mechanism: the journal's coverage losses (honest "not in medium")
  should drop measurably without fabrications appearing.
- Reject any mechanism whose captured facts fail claim-modeling Rule 7
  spot-checks (would you blame it in a week?).

## Owner decision (2026-07-04)

Pilot **1 + 3**; mechanism 2 (LLM-in-the-loop) rejected for now — it
dilutes the measured differentiator ("the journal holds only what the
agent vouched for", the anti-confabulation property from COG-041).

Shipped for the pilot: selective event capture as the hook's default
(git commits + suite status, supersede semantics, `COGIT_CAPTURE=all`
keeps the firehose), MCP `record` batch tool, hooks wired locally in the
cogit repo (`.claude/settings.local.json`, untracked). Measurement gate
stands: re-run the COG-041 real-history track after ~2 weeks — coverage
losses should drop without fabrications appearing.
