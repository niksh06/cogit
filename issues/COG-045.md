# COG-045: Calibration and volatility analytics over the journal

Priority: P1 (owner request: bring measurable math to the journal;
quantifies the trust differentiator from COG-041)

## Problem

The journal accumulates exactly the data needed to answer "whose
confidence should I trust" — assertions with confidence_bps plus their
eventual fate — and no tool reads it. Volatile knowledge areas (values
superseded again and again) are likewise invisible until they bite.

## Decisions

- Outcomes are inferred STRUCTURALLY from history, never from message
  text: for each thought, removed = parents' mindsets minus own;
  a removed assertion is `refuted` when the same thought activates a
  claim negating its claim, `superseded` when the same thought adds a
  belief of the same claim family (same subject/predicate/qualifiers,
  different object — or a rival on the same claim), `retired` otherwise;
  assertions still active at the ref are `open`.
- **Calibration report**: per confidence band (cookbook bands: observed
  >=9800, stated 9000..9799, inferred 7000..8999, hypothesis 4000..6999,
  speculation <4000) and per source type: n / open / superseded /
  refuted / retired, avg confidence, survival = open/(open+refuted).
  Superseded and retired are value churn, not epistemic failure — they
  are excluded from survival and reported separately.
- **Volatility report**: per claim family — revision count, refutation
  count, current value, last change; top-N by revisions.
- Placement: `prototype/integrations/analytics.py` (standalone CLI +
  MCP tool `analytics`), read-only, public porcelain only. Promotion to
  the core CLI contract (with Rust parity) is deferred until the report
  shape survives real use — same staging as the viewer took.

## Acceptance criteria

- On a repo with known lifecycle fixtures the classifier assigns
  open/superseded/refuted/retired exactly; bands and survival compute
  correctly; volatility orders families by revisions.
- MCP tool returns the same document; `--json` and text output; tests
  cover classification, bands, volatility ordering, empty repo.
