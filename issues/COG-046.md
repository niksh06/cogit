# COG-046: ADR-0013 — derivation edges (premises) between beliefs

Priority: P2 (strategic; unlocks the graph-math tier and feeds the
trust line COG-026)

## Problem

Assertions carry external provenance (`source`) but not intra-journal
derivation: nothing records that belief B was inferred FROM beliefs
A1..An. Without those edges the journal cannot answer the questions
graph algorithms are good at: what is tainted if source X was poisoned
(reachability), how strong is the support chain of a conclusion
(bottleneck/maximin path — the meaningful Dijkstra relative), which
beliefs are load-bearing (centrality).

## Scope of the ADR (to write, not to implement here)

- Additive `premises: [assertion-ids]` field on assertions (object
  format v1 stays valid: absent field == no recorded premises); identity
  impact (premises are part of the assertion preimage — two derivations
  of the same claim from different evidence are different assertions —
  intended).
- Write-side capture: `add_fact --premise <id>` / MCP arg; the coverage
  question (agents must actually record premises) inherits COG-044's
  pilot-and-measure discipline.
- Read-side algorithms staged behind usage: `taint <assertion-or-source>`
  (reachability closure), `support <assertion>` (maximin confidence
  path to leaf evidence), criticality later.
- Explicit non-goals: automatic belief fusion (Dempster-Shafer,
  Bayesian networks over rivals) — fusing confidences erases authorship
  and dilutes the anti-confabulation property.

## Acceptance criteria

- ADR-0013 written with the identity/compat analysis and the staged
  algorithm plan; vectors impact assessed (additive object change needs
  new golden vectors per CQ-011 process). Implementation tickets spawn
  from the ADR, not from this ticket.
