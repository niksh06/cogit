# COG-049: Premises v1 — format, validation, vectors, write surface

Priority: P2 (first implementation slice of ADR-0013)

## Scope

- Object format: optional `premises: [assertion-ids]` on assertions —
  sorted, deduplicated, every ID must resolve to an existing assertion
  (write-time check in both runtimes). DAG by construction (see ADR).
- One additive golden vector (assertion with premises); frozen vectors
  intact (CQ-011 process, as ADR-0012 did).
- Write surface: `add-fact --premise <id>` repeatable; MCP
  `add_fact`/`record` gain `premises`; micro-commit path included.
- Read exposure only (no algorithms yet): `premises` in fact rows,
  `show`, `dump`; viewer fact panel lists them as links.
- Interop step: python writes a derived belief with premises, rust
  reads them back (and vice versa).

## Acceptance criteria

- Both runtimes validate premises identically (missing/wrong-type IDs
  rejected with the same error class); vectors reproduce byte-for-byte;
  rows/dump expose the field; interop step green.
