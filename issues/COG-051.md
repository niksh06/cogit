# COG-051: Persistent viewer deployment + bookmarkable project views

Priority: P1 (owner request: an always-on web view of the shared journal)

## Decisions

- `deploy/Dockerfile` + `deploy/docker-compose.yml`: stock
  `python:3.13-alpine` plus the prototype tree (zero dependencies pays
  off — nothing to install), journal bind-mounted **read-only** so the
  viewer's read-only-by-construction guarantee becomes a
  filesystem-level one, container root fs `read_only: true`, port
  published to `127.0.0.1` only (no auth by design), `restart:
  unless-stopped` for persistence across reboots.
- Live updates need nothing special: the page polls `/api/state` every
  3 s and the bind mount exposes journal writes immediately (objects are
  immutable files; refs flip via rename, so readers never see torn
  state).
- Per-project views: `/?project=<slug>` preselects the beliefs project
  filter — bookmarkable per-project URLs over one shared journal.

## Acceptance criteria

- `docker compose up -d --build` serves the live journal on
  `127.0.0.1:8323`; container survives restarts; journal mount is `:ro`.
- `/?project=X` opens with the filter applied when project X exists.
