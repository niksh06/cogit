# COG-052: Writer attribution — expose actor, per-session defaults, Rule 10

Priority: P1 (owner audit question: "can we tell WHO wrote this?" —
measured answer was no: 82 of 87 live assertions said actor="agent")

## Problem

Identity slots exist at every level (assertion.actor, thought.author,
anchor/annotation.author, reflog actor) but attribution failed three
ways: everything defaulted to the same generic string, session identity
leaked into source.uri as ad-hoc self-declarations, and the porcelain
did not even expose actor in fact rows — the recorded identity was
invisible without cat-object.

## Decisions

- **Expose**: fact rows carry `actor` in both runtimes (facts/show/
  dump/recap rows, viewer fact panel); interop asserts the round-trip.
- **Default identity without discipline**: the MCP server generates a
  per-process instance id (`agent-<8hex>`, override `COGIT_ACTOR`) and
  uses it wherever "agent" was the default — one server process serves
  one Claude Code session, so parallel sessions become distinguishable
  automatically. The capture hook signs as `claude-code-<session8>`
  using the session id Claude Code passes in hook payloads.
- **Convention**: cookbook Rule 10 — actor = WHO wrote (stable instance
  id), source = WHERE the information came from; never plain "agent"
  (lint R10-generic-actor, info level).
- **Explicit non-goal**: unforgeable identity — local journals trust
  the filesystem user; signatures arrive with sharing (COG-026).

## Acceptance criteria

- Rows expose actor in both runtimes; MCP defaults produce
  `agent-<id>` (never bare "agent"); hook writes carry the session id;
  lint flags generic actors; interop actor round-trip green.
