# COG-054: viewer v2 — work made visible (lanes, actors, project threads) + UX pass

Priority: P1 (owner requests 2026-07-09: "длинные ответы обрезаются — пусть
выпадают шторкой", "улучшим интерфейс согласно best practices", and "в гите
я привык видеть ветки как олицетворение работы — тут нет разветвления, нет
олицетворения работы агентов")

## Scope (owner-approved; sims scene explicitly REJECTED, see journal
`cogit:COG-054 scope`)

1. **Structure of work in the DAG** — the git-graph feel:
   - each graph lane gets a stable color; fork/merge edges take the color
     of the branch-side lane, so divergence reads at a glance;
   - merge nodes render larger with an outline;
   - every thought row carries its writer's avatar (initials, color from
     the actor hash — COG-052 made writers distinguishable) and a colored
     author name;
   - an actor legend above the DAG (avatar + name + thought count);
     clicking an actor dims everything they did not write;
   - thought rows carry project chips (`build_state` now emits per-node
     `projects` from the project qualifier of added/removed beliefs);
     selecting a project — via chip, filter, or `?project=` — dims all
     thoughts outside that project's thread.
2. **Шторка for long texts**: belief-table values longer than 60 chars (or
   multi-line) get a chevron that expands a full-width `pre-wrap` sub-row;
   expansion state survives re-render/polling. DAG rows stay fixed-height
   (SVG rail alignment — the trap from the handoff): their full text lives
   in the detail panel, which now renders `pre-wrap` too.
3. **UX pass**: Esc closes the detail panel; ids in the detail panel copy
   on click with a toast; relative timestamps ("14h ago", absolute in
   title, refreshed while polling; snapshots keep absolute); all four
   filters sync to the URL (bookmarkable filtered views); visible focus
   styles; empty-filter state message; scroll position preserved across
   poll re-renders; subtle transitions.
4. **Activity strip**: folded INTO the DAG as the actor legend + per-row
   avatars instead of a separate ticker (a ticker would duplicate the
   thought list one card above it).

## Non-goals

- Sims-like animated scene — rejected by owner (wrong genre for an audit
  tool; the concept moved to another project).
- Canvas/animation of any kind; the viewer stays a zero-dependency page.

## Acceptance criteria

- Two-lane fixture renders two distinct lane colors and one merge dot.
- `build_state` nodes carry sorted `projects`; thoughts with no
  project-qualified beliefs carry `[]`.
- Live check on the shared journal: expand/collapse, actor dim, project
  chip → filter + URL, Esc, copy toast, URL preselect on reload, console
  clean. Container rebuilt.
