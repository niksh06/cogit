# COG-056: Atomic lifecycle porcelain for active beliefs

Priority: P0 after COG-055 (live shared journals accumulate stale active state
because the parallel-safe `add_fact(commit=true)` path can only add; a correct
supersede/refute requires several staged operations)

## Problem

Cogit already has the right primitives: explicit `remove-fact --reason`,
`claim.negates`, invariant 25, and structural lifecycle analytics. The safe,
one-call path used by parallel agents does not expose them as one atomic state
transition.

The result is predictable: agents add “shipped” while “awaiting approval”
remains active, write a correction without retiring the corrected assertion,
or describe a REFUTE in prose without recording a structural refutation. The
Aleph journal audit found 125 active assertions and zero active negations, with
several such stale pairs.

## Decisions (proposed)

- Add explicit one-call lifecycle operations:
  - `supersede-fact`: remove an active target with reason `superseded` and add
    a replacement in the same claim family;
  - `refute-fact`: remove all active assertions of the target claim with reason
    `refuted` and activate a claim whose `negates` points to it;
  - `retire-fact`: remove explicit active targets with a required reason,
    without asserting falsity.
- Target assertion IDs are mandatory. The engine never guesses from prose,
  recency or semantic similarity.
- A superseding replacement preserves `kind`, `subject`, `predicate` and
  qualifiers by default; changing the family requires a separate explicit
  record operation.
- Publish the transition as one thought via the COG-035 CAS/retry model,
  without the shared index. If the target is no longer active, return a
  stale-target result; do not silently apply against a new state.
- Surface the operation in Python and Rust CLIs, MCP, docs and interop tests.

## Non-goals

- Automatic truth arbitration between rival sources.
- Implicit expiry based on age.
- Treating a replacement value as a negation of the old value.

## Acceptance criteria

- Each operation is one thought and exposes old/new assertion IDs, affected
  claim IDs and the thought ID.
- `supersede-fact` leaves only the replacement active and analytics classifies
  the target as `superseded`.
- `refute-fact` removes every active assertion of the negated claim, activates
  exactly one explicit negation and satisfies invariant 25.
- Inactive target, family drift, invalid replacement, dirty index or failed
  validation leave HEAD/index unchanged.
- Two writers racing to transition the same target produce one winner; the
  other receives a stale-target error. No mindset contains both replacements
  by accident.
- Python, Rust, MCP and interop fixtures produce the same active mindset and
  lifecycle classification.
