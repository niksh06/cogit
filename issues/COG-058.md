# COG-058: Lint v2 + lifecycle hygiene ratchet

Priority: P1 (live-journal evidence: the Aleph slice had 125 active facts,
234 warnings and stale lifecycle descriptions that the current shape-only
linter cannot turn into an actionable cleanup queue)

## Problem

COG-047 correctly detects prose objects, unstable subjects, blob qualifiers,
confidence-band mismatches and missing project qualifiers. On an established
journal, a flat list of hundreds of findings is hard to adopt: `--strict`
cannot distinguish old debt from regressions, MCP output is unbounded, and
there is no structural view of competing active state.

Some stale cases are also modeled so differently that exact claim-family
analytics cannot connect them: “pending” and later “REFUTE” may live in prose
or identity-bearing qualifiers, while a correction may use a new subject.
Cogit must surface candidates without pretending it can infer truth.

## Decisions (proposed)

- Add a baseline ratchet: `lint --since <anchor|ref>` classifies active
  findings as `existing` or `new`; `--strict --since` fails only on new
  warnings. Existing `--strict` semantics remain unchanged.
- Add bounded output controls: `--summary`, `--limit`, `--rule` and
  `--severity`, mirrored by MCP with safe defaults.
- Add read-only lifecycle hygiene candidates:
  - different active objects in the same stable family
    `(kind, subject, predicate, qualifiers)`;
  - singleton-state collisions for known state predicates such as `status`,
    `owner` and `priority`;
  - advisory markers such as correction/supersede/REFUTE/pending without a
    corresponding structural transition, clearly labelled heuristic.
- Multiple assertions of the same claim/object are corroboration, not stale
  duplicates.
- Every candidate includes assertion/claim IDs, actor, source, introducer,
  confidence and the relevant lifecycle history. Age is only a review hint,
  never evidence of falsity.
- No autofix. Remediation is an explicit COG-056 transition or a reviewed
  batch record.

## Dogfood migration

After COG-055/056 ship, run the tool against the shared journal by project.
Create a safety anchor and pre-clean JSON snapshots, assign every high-signal
candidate a `fixed` or `intentional` disposition, repair append-only, then
create a post-clean anchor and lint baseline. Do not delete objects, rewrite
history or expire reflogs.

## Acceptance criteria

- Old active debt before the baseline is `existing`; a new violation after it
  is `new`. Superseding the old violation removes it from active debt, and a
  later reintroduction is new.
- Missing/non-ancestor baselines fail clearly; `--project` scopes findings and
  thought history.
- Exact-family rivalry is detected; same-claim corroboration is not.
- Fixtures exercise awaiting->shipped, correction-without-retirement and
  pending->REFUTE cases. Heuristic findings never mutate state and are labelled
  separately from structural findings.
- MCP output is bounded and reports full totals by rule/severity even when
  detail rows are limited.
- The live cleanup leaves `verify` healthy, no unreviewed singleton-state
  collisions, and no new lint warnings after the baseline; before/after
  artifacts and anchors are recorded.
