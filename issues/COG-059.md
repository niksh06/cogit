# COG-059: Project-scoped compact reader + health surface

Priority: P1 (one shared-journal re-anchor still leaks cross-project thought
log and can return tens of thousands of tokens; analytics has no project
filter, so a project health check requires manual multi-call archaeology)

## Problem

COG-053 scopes recap rows and thoughts, but `dump(project=...)` still returns a
global recent log. `analytics` has no project argument. Fact rows omit
`asserted_at` and `method`, so freshness and provenance review require raw
object reads. Full fact objects also make a one-call dump expensive on a
journal with long prose debt.

The Aleph audit needed separate `status`, `dump`, `recap`, `facts`, `lint`,
Git and manual summarization. The unfiltered dump exceeded 76k tokens.

## Decisions (proposed)

- Complete project parity:
  - `analytics --project` classifies only assertions and thoughts that touch
    that project;
  - `dump --project` scopes its log as well as facts and recap;
  - fact rows expose `asserted_at` and `method` in Python/Rust porcelain.
- Add a backward-compatible compact mode. It preserves IDs, subject,
  predicate, typed scalar object, negation, confidence, actor/source and
  lifecycle counts, while replacing long prose/blob payloads with bounded
  previews and byte/word counts. Full mode is unchanged.
- Add a one-call project health document over existing read-only surfaces:
  repository integrity status, active/revised/refuted/negated counts, lint
  totals and new-debt totals, lifecycle candidates, last project thought and
  relevant anchor. Detail lists are top-N bounded but totals are exact.
- Require `project` for the compact shared-journal health path unless the
  journal contains only one project; never silently mix projects.

## Non-goals

- Retrieval, semantic summaries or LLM-generated health judgments.
- Hiding the IDs needed to inspect the full immutable objects.
- Changing full `dump` output without an explicit mode.

## Acceptance criteria

- In a two-project fixture, scoped analytics, dump log, recap and health contain
  no foreign project thoughts or assertions.
- Python/Rust fact rows expose identical `asserted_at` and `method` values;
  interop is green.
- A synthetic project with 125 long facts produces a compact health artifact
  below 20 KB while full mode remains byte-for-byte compatible in shape.
- Health reports active negations, revised-family ratio, exact lint totals and
  bounded candidate details with assertion IDs.
- Unknown project returns an empty scoped result with explicit project name,
  not a global fallback.
- MCP schemas and descriptions identify compact health as the preferred
  re-anchor for journal quality; `dump` remains the full-state reader.
