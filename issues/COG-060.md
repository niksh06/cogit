# COG-060: Viewer lifecycle and journal-health lens

Priority: P2 after COG-058/059 (possible product feature; read-only diagnosis
has proved useful, while polling optimization must first meet the roadmap's
measured-pressure entry criterion)

## Problem

Viewer v2 makes branches, writers and project threads visible, but a human
still cannot see whether the active belief-state is healthy: lint debt,
competing current values, absence of structural lifecycle transitions and
review candidates are available only through separate tools.

The live page also rebuilds the full state and transmits it every three seconds.
That may become wasteful as journals grow, but the roadmap forbids speculative
performance layers without a measured threshold.

## Decisions (proposed)

- Add a project-scoped, read-only health panel backed by COG-059:
  active facts, structural negations, revisions/refutations, existing vs new
  lint debt, lifecycle candidates and last relevant anchor/thought.
- Add a lifecycle lens in the beliefs table: group exact families, distinguish
  corroboration from competing values, show introducer/actor/source and copy
  the explicit COG-056 remediation command or payload.
- Preserve the viewer's read-only guarantee. There are no mutate buttons and
  no browser-supplied actor identity.
- Measure `/api/state` rebuild time, payload size and unchanged-poll traffic on
  the real shared journal plus a synthetic 10x fixture. Only if the documented
  threshold is crossed, add a state fingerprint/ETag and return 304 for an
  unchanged poll without rebuilding the expensive body.
- Snapshot mode renders the same health data without live-only controls.

## Non-goals

- Automatic cleanup or truth arbitration.
- Implicit expiry of old beliefs.
- Semantic search or an LLM-generated journal summary.
- Performance work without measured pressure.

## Acceptance criteria

- A fixture with active/corroborating/competing/refuted beliefs renders the
  correct counts and evidence IDs; project switching fully re-scopes health.
- Existing and new lint debt are visually distinct, and long detail remains
  expandable rather than flooding the main table.
- Suggested remediation always names explicit assertion IDs; the viewer never
  issues a mutating HTTP method (pinned by test).
- Snapshot and live state agree for the same ref/project.
- A benchmark artifact records rebuild latency, payload bytes and unchanged
  traffic at current and 10x scale, plus the adopt/no-adopt threshold verdict.
- If the gate adopts ETag: a second unchanged poll returns 304 and does not
  rebuild `build_state`; any ref/annotation change invalidates the fingerprint.
