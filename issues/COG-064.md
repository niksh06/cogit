# COG-064: one-call nuance — `detail` becomes an annotation

Priority: P2 (field feedback: R2 forces compressing rich verdicts into
dash-tokens, so the rich version lived outside the journal in FACTS.md)

## Resolution

MCP `add_fact` (both flows) and `record` fact entries accept `detail`:
a rich prose string that lands as an annotation (namespace `detail`) on
the assertion in the SAME tool call. The claim object stays an atomic
value; the nuance stays in the journal, next to the fact, rendered by
the existing annotation surfaces (show/viewer). Lint R2's message now
points to it. Annotation publication is auxiliary (not atomic with the
thought) — a crash between the two loses only the note, never the fact.

## Acceptance criteria

- Minimal fact + detail in one MCP call: fact active, annotation
  readable via `annotations`, `detail` visible in the viewer detail
  panel (existing annotations block).
- Empty/non-string detail is a clean tool error.
