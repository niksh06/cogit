# COG-067: discipline becomes a feedback loop — write-time hints + digest debt line

Priority: P1 (inventory 2026-07-11: 41% prose objects while lint is
pull-only — an agent that never calls lint never learns; the gap between
"wrote a bad fact" and "found out" was a whole epic)

## Resolution

- `add_fact` / `record` MCP responses gain `hints`: the just-written
  row(s) run through the linter's `check_row` and any findings come back
  IN the tool result ("R2-prose-object: 24 words — keep the nuance by
  passing detail="). Advisory: the fact already landed; the very next
  call gets written better. Clean writes add no key (lean responses).
- The SessionStart digest gains a ratchet line: with a `lint-baseline-*`
  anchor present, the digest reports "no new debt since <baseline>" or
  "N NEW finding(s) … fix with supersede_fact/retire_fact" — the ratchet
  reaches every session without being asked. Digest failures never break
  a session.

## Acceptance criteria

- Prose object via MCP → response carries an R2 hint; clean fact →
  no `hints` key; batch `record` aggregates hints across facts.
- Digest with baseline + clean journal says "no new debt"; after a new
  violation it names the count and the remediation ops; without a
  baseline anchor the line is absent.
