# COG-069: Engram bridge — semantic layer OVER the journal (stub)

Priority: P3 (idea parked by owner request 2026-07-11; entry criterion:
a real "find by meaning" miss that `search` (COG-068) cannot cover)

## Problem

`search` is deliberate substring grep (ADR-0002: no embeddings in
cogit). Questions like "что мы решали про деградацию качества алертов?"
need a semantic layer. The owner already runs one: Engram (RAG hub with
ingest watcher).

## Sketch (not committed)

- An exporter (integrations-level) renders the journal per project into
  OKF-fronted markdown snapshots — active beliefs + detail annotations +
  lifecycle history — into `~/Reports/projects/<project>/journal/`;
  the existing engram-sync watcher ingests them.
- Cogit stays the source of truth and keeps ids in the exported text, so
  a semantic hit routes BACK to `blame_fact`/`premises` by assertion id.
- Refresh policy: on anchor creation or nightly; no daemon inside cogit.

## Non-goals

- Embeddings, ranking or retrieval inside cogit itself (ADR-0002).
- Two-way sync: Engram never writes back to the journal.
