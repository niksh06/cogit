# COG-070: version discipline — CHANGELOG as the source of truth

Priority: P1 (owner decision 2026-07-12: "помечать артефакты версией
cogit; версии фиксировать в CHANGELOG.md")

## Problem

Cogit is in active field use while under active development, and the
tool has no working notion of its own version:

- `__version__` / Cargo `version` have said `0.1.0` since day one —
  they carry no information and are not checked against each other.
- Nothing user-visible reports a version, so version skew is invisible
  until it bites. It bit twice in one week: a stale MCP server threw
  `CorruptionError: unknown fields: ['removals']` (pre-ADR-0015 reader
  x newer writer), and a stale viewer container ran the pre-COG-061
  quadratic `build_state` (124.6 s polls) with no way to see it was old.
- The repository FORMAT is already versioned (`repositoryFormatVersion`
  in `.cogit/config`) — this ticket is about the SOFTWARE version.

## Resolution

1. **CHANGELOG.md** at the repo root (Keep a Changelog shape, semver
   0.x): the topmost release heading IS the current version. 0.1.0 is
   recorded retroactively (2026-07-02 MVP); 0.2.0 collects everything
   shipped since publication.
2. **One version, three copies, one ratchet**: `prototype/cogit/
   __init__.py`, both `Cargo.toml`s, and the CHANGELOG top heading must
   agree — enforced by an interop-test step (`--version` of both
   runtimes vs CHANGELOG), so drift fails CI, not a user.
3. **`--version`** on both CLIs, output `cogit <semver> (<impl>)` where
   impl is `py` | `rs`.
4. **Surfacing where skew hurt**: viewer footer + `/api/health` and the
   `health` integration (`reader` field), MCP `serverInfo` (already
   wired to `__version__`), SessionStart digest first line.

## Acceptance criteria

- `python3 -m cogit --version` and `cogit --version` print the same
  semver; interop step fails if either disagrees with CHANGELOG.md.
- `health()` output names the reader build (`cogit-py/<v>`).
- Viewer page shows the version (footer); container rebuild documented.
- SessionStart digest names the hook's cogit version.

## Non-goals

- Stamping artifacts (objects) with the writer version — that is
  COG-071 / ADR-0016.
- Build/commit hashes in the version string — allowed by the format
  later (`+<build>`), not emitted now (would need git introspection in
  two zero-dependency runtimes for unproven value).
